# 🏥 Proyecto Web - Hospital Regional Eleazar Guzmán Barrón

## 🎯 Objetivo del Proyecto
Diseñar, estructurar y optimizar la plataforma web informativa y de servicios para el Hospital Regional Eleazar Guzmán Barrón. El proyecto no solo replica la identidad institucional, sino que **propone mejoras significativas en la Experiencia de Usuario (UX/UI) e interactividad** en comparación con la web oficial, aplicando un flujo de desarrollo frontend moderno y control de versiones colaborativo y limpio.

## 🚀 Funcionalidades Premium e Interactividad (Fase 2)
A diferencia de un sitio estático tradicional, esta versión incorpora lógica avanzada en el lado del cliente (Frontend):
* **Motor de Animación Global (Scroll Reveal):** Revelación progresiva de secciones mediante el uso de `IntersectionObserver` en JavaScript, evitando la saturación visual.
* **Buscador de Convocatorias en Tiempo Real:** Tabla interactiva con filtrado dinámico instantáneo de procesos CAS.
* **Búsqueda Inteligente de Especialidades:** Filtrado dinámico por síntomas o palabras clave orientada al paciente.
* **Formulario de Citas con Selectores en Cascada:** Validación estricta de DNI e inyección dinámica de personal médico según la especialidad elegida.
* **Portal del Paciente y Transparencia en Modales:** Sustitución de formularios invasivos por ventanas flotantes emergentes fijas (`position: fixed`).
* **Sistema Global de Notificaciones:** Integración estética y profesional mediante la librería **SweetAlert2**.

## 🛠️ Tecnologías Utilizadas
* **HTML5:** Maquetación semántica y estructuración del DOM.
* **CSS3:** Diseño responsivo (Flexbox/Grid), variables globales, estilos de tablas modernas y transiciones de animación fluidas.
* **JavaScript (ES6+):** Manipulación dinámica del DOM, manejo de arrays/objetos de datos, eventos de teclado/cambio y API de `IntersectionObserver`.
* **SweetAlert2:** Librería externa para el manejo elegante de alertas y respuestas del sistema.
* **Git & GitHub Flow:** Gestión estricta de ramas de funcionalidad (`feat-js-`), resolución limpia de conflictos y despliegue automatizado en GitHub Pages.
